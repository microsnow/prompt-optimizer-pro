# Web 部署前检查清单

## 📋 部署前准备

### 构建和依赖
- [ ] 运行 `npm install` 安装所有依赖
- [ ] 运行 `npm run build` 构建前端（检查无报错）
- [ ] 检查 `dist/` 文件夹已生成
- [ ] 文件大小合理（通常 < 1MB JS）

### 代码检查
- [ ] 移除所有 `console.log` 和调试代码
- [ ] 检查没有硬编码的 API Key
- [ ] 检查没有本地 `localhost` 地址
- [ ] 验证 API 端点使用相对路径（`/api`）

### 配置检查
- [ ] 创建 `.env.production` 文件
- [ ] 配置 `ALLOWED_ORIGINS` 环境变量
- [ ] 配置 `NODE_ENV=production`
- [ ] 检查 `LOG_LEVEL` 设置为 `info` 或 `warn`

---

## 🔐 安全检查

### API 密钥安全
- [ ] API Key **不存储**在前端代码中
- [ ] API Key **不存储**在 Git 仓库中
- [ ] 使用 `.gitignore` 排除 `.env.*` 文件
- [ ] 生产环境密钥通过环境变量或密钥管理系统传入

### CORS 安全
- [ ] `ALLOWED_ORIGINS` 只包含受信任的域名
- [ ] 移除 `file://` 协议（仅用于开发）
- [ ] 移除 `localhost` 地址（仅用于开发）
- [ ] 测试跨域请求被正确拒绝

### HTTPS 安全
- [ ] 配置有效的 SSL/TLS 证书
- [ ] 启用 HSTS 头部（强制 HTTPS）
- [ ] 禁用不安全的协议（TLSv1.0, TLSv1.1）

### 请求验证
- [ ] 验证请求内容类型
- [ ] 检查请求体大小限制
- [ ] 实施速率限制
- [ ] 添加请求日志和监控

---

## 🚀 服务器部署

### 前端资源
- [ ] 前端文件通过 Web 服务器（Nginx/Apache）提供
- [ ] 配置 gzip 压缩
- [ ] 配置浏览器缓存策略
- [ ] 配置 SPA 路由（Vue Router 支持）

### API 代理
- [ ] Node.js 代理服务器已安装依赖
- [ ] 可以启动代理服务：`npm run api:proxy`
- [ ] 代理监听端口 3001（或配置的端口）
- [ ] 代理进程在后台运行（PM2/systemd/Docker）

### 反向代理
- [ ] Nginx 配置已编写
- [ ] 前端请求 → 静态文件或 index.html
- [ ] `/api` 请求 → 转发到 Node.js 3001 端口
- [ ] 测试所有请求路由正确

### 进程管理
- [ ] 使用 PM2 或 systemd 管理 Node.js 进程
- [ ] 配置进程自动重启
- [ ] 配置日志轮转
- [ ] 配置开机自启

## 🎯 常见问题快速排查

| 问题 | 可能原因 | 解决方案 |
|-----|--------|--------|
| 无法访问网站 | DNS/防火墙 | 检查 DNS 设置，检查防火墙规则 |
| CORS 错误 | `ALLOWED_ORIGINS` 配置错误 | 检查环境变量，重启代理 |
| API 超时 | 代理服务未启动或网络问题 | 检查 Node.js 进程，测试网络 |
| 页面空白 | 前端构建失败或路由问题 | 检查 dist 文件夹，检查 Nginx 配置 |
| 内存泄漏 | Node.js 进程长时间运行 | 使用 PM2 配置重启，分析内存使用 |
